<objective>
Build the match execution pipeline: database schema for matches, MatchSandbox (reusable isolate), match runner service, Redis pub/sub broadcaster, BullMQ job queue, and match API endpoints.

Purpose: This connects the Snake engine (Plan 01) to the API layer so users can start matches and retrieve results. Frames are recorded for replay (SPEC-02) and broadcast for live spectating (SPEC-01, wired in Plan 03).

Output: Working match execution from API request to stored result. User can POST to start a match, match runs asynchronously, frames stored in PostgreSQL, GET returns match data with replay frames.
</objective>

       - `matches` pgTable with columns: id (text, PK, nanoid), userId (text, FK to user.id), agentId (text, FK to agents.id), discipline (text, default "snake"), seed (text), config (jsonb), status (text, default "queued"), score (integer), ticksPlayed (integer), terminationReason (text), frames (jsonb), errorMessage (text), createdAt (timestamp, defaultNow), startedAt (timestamp), completedAt (timestamp)

    4. Create `packages/engine/src/match-sandbox.ts`:
       - `MatchSandbox` class that creates ONE isolated-vm isolate at construction
       - Constructor takes compiled JS code string and SandboxOptions
       - `init()` async method: creates context, cleans globals, compiles agent script once
       - `callMove(state: unknown)` async method: injects state, calls move(), returns SandboxResult
       - `dispose()` method: disposes isolate
       - Reuses the same isolate/context/script across calls (critical for performance -- see Pitfall 4 in research)
       - Same global cleanup as existing sandbox.ts (delete console, setTimeout, etc.)

    5. Create `packages/api/src/services/match-broadcaster.ts`:
       - `createMatchBroadcaster(redisUrl: string)` factory returning publisher
       - `publishFrame(matchId, frame)`: publishes to `match:{matchId}` Redis channel
       - `publishMatchEnd(matchId, result)`: publishes match_end event
       - If REDIS_URL is not set, create a no-op broadcaster (graceful degradation for tests without Redis)

    6. Create `packages/api/src/services/match-runner.ts`:
       - `runMatch(agentCompiledCode, config, options?)` async function
       - Creates SnakeEngine from Plan 01's `createSnakeEngine`
       - Creates MatchSandbox with the agent code
       - Loop: while alive and tick < maxTicks, get state, call sandbox.callMove(state), validate move with snakeMoveSchema.safeParse(), apply move to engine, record frame, optionally broadcast via match-broadcaster
       - On agent error (timeout, crash): set terminationReason to "crash" or "timeout", stop loop
       - Returns MatchResult: { matchId, frames, finalScore, ticksPlayed, terminationReason }
       - Dispose sandbox in finally block

    7. Update `packages/engine/src/index.ts` to export MatchSandbox.

    8. Write tests FIRST:
