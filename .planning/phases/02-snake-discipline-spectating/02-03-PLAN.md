  <action>
    1. Create `packages/api/src/routes/spectate.ts`:
       - `createSpectateRoutes(upgradeWebSocket)` factory function
       - Route: `/matches/:matchId/spectate` -- upgrades to WebSocket
       - onOpen: Create a Redis subscriber (ioredis), subscribe to `match:{matchId}` channel. On each message, parse JSON and send to WebSocket client
       - onClose: Unsubscribe from Redis channel, quit subscriber
       - onMessage: Handle client commands (future: pause/resume, but for now just ignore)
       - Each WebSocket connection gets its own Redis subscriber instance (ioredis requires separate connections for subscribe mode)

    2. Update `packages/api/src/index.ts`:
       - Import `createNodeWebSocket` from `@hono/node-ws`
       - Create `{ injectWebSocket, upgradeWebSocket }` from `createNodeWebSocket({ app })`
       - CRITICAL: Move CORS middleware to `/api/*` only (not `*`) to avoid CORS/WebSocket header conflict (see Pitfall 3 in research)
       - Mount spectate routes under `/ws` path: `app.route("/ws", createSpectateRoutes(upgradeWebSocket))`
       - After `serve()`, call `injectWebSocket(server)` to enable WebSocket upgrades
       - Keep logger middleware on all routes

    3. Create `packages/api/src/__tests__/spectate-ws.test.ts`:
       - Test: WebSocket connection to `/ws/matches/:matchId/spectate` succeeds (may need to test at integration level or mock)
       - Test: Published frame on Redis channel is received by WebSocket client
       - Note: WebSocket testing may require starting the server. If too complex for unit test, create a minimal test that verifies the route setup and mock the Redis subscriber

    4. Create `packages/api/src/__tests__/replay.test.ts`:
       - Test: GET /api/matches/:id for completed match returns frames array
       - Test: GET /api/matches/:id for non-existent match returns 404

    2. Create `apps/web/src/components/game/snake-renderer.tsx` (client-only component):
       - "use client" directive
       - Props: `frames: SnakeFrame[]`, `currentFrameIndex: number`, `showOverlay: boolean`
       - useEffect: Create PixiJS 8 Application with async `app.init()` (see research Pattern 4)
       - Canvas size: 600x600, dark background (0x1a1a2e)
       - Render grid lines (subtle gray), snake body (green rectangles), snake head (brighter green), food (red circle)
       - On currentFrameIndex change, update graphics positions from `frames[currentFrameIndex].state`
       - Cell size = canvas width / gridWidth (integer division)
       - Cleanup: app.destroy(true) on unmount
       - Do NOT import at module level in any SSR-capable file

    3. Create `apps/web/src/components/game/decision-overlay.tsx`:
       - "use client" directive
       - Props: `agentMove: string | null`, `agentReasoning: string | null`, `tick: number`, `score: number`
       - Renders a semi-transparent panel overlaid on the game canvas (absolute positioned)
       - Shows: current tick, score, last agent move direction, agent reasoning text
       - Uses Tailwind for styling (bg-black/70, text-white, rounded, etc.)
       - Satisfies SPEC-03: "Decision overlay shows high-level agent decisions during spectating"

    4. Create `apps/web/src/components/game/replay-controls.tsx`:
       - "use client" directive
       - Props: `totalFrames: number`, `currentFrame: number`, `isPlaying: boolean`, `onPlay: () => void`, `onPause: () => void`, `onStep: (delta: number) => void`, `onSeek: (frame: number) => void`
       - Play/Pause button, Step forward/backward buttons, Frame scrubber (range input)
       - Shows current frame / total frames counter

    5. Create `apps/web/src/components/game/game-canvas.tsx`:
       - "use client" directive
       - Dynamic import of SnakeRenderer with `{ ssr: false }` via `next/dynamic`
       - Manages frame playback state: currentFrameIndex, isPlaying, playback interval
       - Two modes:
         - `mode: "replay"`: Loads frames from props, controls playback locally
         - `mode: "live"`: Connects to WebSocket at `ws://API_URL/ws/matches/{matchId}/spectate`, appends incoming frames, auto-advances currentFrameIndex
       - Renders SnakeRenderer, DecisionOverlay, and ReplayControls
       - Playback interval: ~200ms per frame (5 fps, matching default tickRate)

    6. Create `apps/web/src/app/matches/[id]/page.tsx`:
       - Fetches match data from API: `GET /api/matches/{id}`
       - If match status is "completed": render GameCanvas in "replay" mode with frames
       - If match status is "running": render GameCanvas in "live" mode with matchId for WebSocket
       - If match status is "queued": show waiting state
       - If match status is "error": show error message
       - Show match metadata: discipline, score, ticks played, termination reason

    7. Create `apps/web/src/app/matches/page.tsx`:

       - "Start New Match" button that POSTs to `/api/matches` with user's first agent (or shows agent selector if multiple)
