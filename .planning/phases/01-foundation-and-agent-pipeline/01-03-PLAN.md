    **packages/engine/src/compiler.ts**:
    - `compileTypeScript(tsCode: string): Promise<string>` -- same esbuild config as agent-compiler.ts in API package (platform: "neutral", format: "esm", target: "es2022").
    - This is the engine-local compiler used for sandbox execution. The API package has its own compiler for upload validation. Both use the same config.

    **packages/engine/src/sandbox.ts**:
    - Define `SandboxOptions`: `{ memoryLimitMb: number, timeoutMs: number }`
    - Define `SandboxResult`: `{ result: unknown, cpuTimeNs: bigint, wallTimeNs: bigint }`
    - `executeInSandbox(compiledJs: string, gameState: unknown, options: SandboxOptions): Promise<SandboxResult>`:
      1. Create `new ivm.Isolate({ memoryLimit: options.memoryLimitMb })` -- use conservative limits (64MB default, per research pitfall about approximate limits)
      2. Create context via `isolate.createContext()`
      3. Inject gameState as a frozen global via `ivm.ExternalCopy`: `await jail.set("__gameState__", new ivm.ExternalCopy(gameState).copyInto())`
      4. Compile agent code: `await isolate.compileScript(compiledJs)`
      5. Run with timeout: `await script.run(context, { timeout: options.timeoutMs })`
      6. Call the agent's `move` function: compile and run `"move(__gameState__)"` with same timeout
      7. Capture `isolate.cpuTime` and `isolate.wallTime` for metrics
      8. **Always** dispose the isolate in a finally block
      9. Return `{ result, cpuTimeNs, wallTimeNs }`
    - **Security**: The context must have NO access to: process, require, console, setTimeout, setInterval, fetch, or any Node.js/browser globals. Only `__gameState__` is injected.
    - Handle errors: timeout -> throw specific TimeoutError, memory -> throw MemoryLimitError, other -> wrap and rethrow

    **Test files**: Use Vitest. Test with real isolated-vm (not mocked). Create test agent code strings inline.
